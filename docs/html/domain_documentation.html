Domain Layer Documentation

Подробная документация доменного слоя

См. domain/domain.dox

Обзор

Domain Layer — это сердце приложения, содержащее всю бизнес-логику. Этот слой полностью независим от технологий, фреймворков и способов доставки данных.

Ответственность Domain Layer

  • Описание бизнес-сущностей и их поведения
  • Реализация бизнес-правил и инвариантов
  • Определение интерфейсов для работы с данными
  • Доменные события и их обработка

Структура Domain Layer

entities/

Описание: Бизнес-сущности с уникальными идентификаторами

Содержит:

  • Entity классы (User, Ticket, TicketMessage и т.д.)
  • Enum для статусов и ролей
  • Методы изменения состояния сущностей
  • Валидация бизнес-правил внутри сущностей

Характеристики Entity:

  • Имеет уникальный идентификатор (id)
  • Может изменять своё состояние
  • Инкапсулирует бизнес-логику работы с собой
  • Две entity равны, если у них одинаковые ID

aggregates/

Описание: Группы связанных entities, обрабатываемые как единое целое

Содержит:

  • Aggregate Root классы
  • Вложенные entities (если агрегат содержит коллекции)
  • Методы управления вложенными объектами
  • Защита границ консистентности

Характеристики Aggregate:

  • Aggregate Root — главная entity для доступа
  • Изменения только через Aggregate Root
  • Гарантирует консистентность данных внутри границ
  • Определяет границы транзакций

value-objects/

Описание: Неизменяемые объекты без идентификатора

Содержит:

  • Value Object классы (Email, Phone, Money и т.д.)
  • Типобезопасные обёртки над примитивами
  • Валидация значений

Характеристики Value Object:

  • Не имеет идентификатора
  • Неизменяемый (immutable)
  • Равенство по значению, а не по ссылке

repositories/

Описание: Интерфейсы для работы с хранилищем данных

Содержит:

  • Интерфейсы Repository (только объявления)
  • Методы для сохранения и загрузки aggregates
  • Методы поиска по критериям

Важно: Только интерфейсы! Реализация в Infrastructure Layer

services/

Описание: Доменная логика между несколькими объектами

Содержит:

  • Domain Service классы
  • Логика, не принадлежащая одной entity
  • Операции над несколькими aggregates
  • Сложные вычисления и алгоритмы

Когда использовать:

  • Операция затрагивает несколько aggregates
  • Логика не является "поведением" одного объекта
  • Сложные бизнес-правила

events/

Описание: События, происходящие в домене

Содержит:

  • Domain Event классы
  • Описания событий (TicketCreated, TicketClosed и т.д.)
  • Интерфейсы обработчиков событий

Правила Domain Layer

  1. Независимость: Не зависит от других слоёв
  2. Чистота: Нет зависимостей от фреймворков, БД, UI
  3. Бизнес-логика: Вся логика здесь, не в сервисах приложения
  4. Инварианты: Всегда защищает свои бизнес-правила
  5. Тестируемость: Легко тестировать без внешних зависимостей

Ключевые концепции

Entity vs Value Object:

  • Entity имеет идентификатор, Value Object — нет
  • Entity изменяема, Value Object — неизменяем
  • Entity сравнивается по ID, Value Object — по значению

Aggregate:

  • Граница консистентности для группы объектов
  • Доступ только через Aggregate Root
  • Определяет границы транзакций

Domain Service:

  • Stateless (без состояния)
  • Работает с несколькими объектами
  • Содержит логику, не влезающую в entity

Repository:

  • Абстракция над хранилищем
  • Работает с целыми aggregates
  • Скрывает детали персистентности
  • Создано системой 1.15.0
