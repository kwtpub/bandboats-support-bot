\chapter{Bandboats Support Bot -\/ DDD Architecture }
\hypertarget{index}{}\label{index}\index{Bandboats Support Bot -\/ DDD Architecture@{Bandboats Support Bot -\/ DDD Architecture}}
\hypertarget{index_intro}{}\doxysection{\texorpdfstring{Введение}{Введение}}\label{index_intro}
Это бот техподдержки, построенный на принципах Domain-\/\+Driven Design (DDD). Архитектура позволяет легко заменять способ взаимодействия с пользователем (Telegram Bot → Web → API) без изменения бизнес-\/логики.\hypertarget{index_architecture}{}\doxysection{\texorpdfstring{Архитектура приложения}{Архитектура приложения}}\label{index_architecture}
Приложение разделено на 4 слоя согласно DDD\+:\hypertarget{index_domain_layer}{}\doxysubsection{\texorpdfstring{1. Domain Layer (Доменный слой)}{1. Domain Layer (Доменный слой)}}\label{index_domain_layer}
{\bfseries{Местоположение\+:}} {\ttfamily src/domain/}

{\bfseries{Ответственность\+:}}
\begin{DoxyItemize}
\item Бизнес-\/логика приложения
\item Сущности (Entities) и агрегаты (Aggregates)
\item Доменные сервисы (Domain Services)
\item Интерфейсы репозиториев (Repository Interfaces)
\item Доменные события (Domain Events)
\end{DoxyItemize}

{\bfseries{Важно\+:}} Этот слой НЕ зависит от других слоёв и технологий!
